웹 스타일 가이드 정의

본 가이드는 HTML마크업을 진행하는 프론트 개발자 들을 대상으로 하며, 프로젝트의 원활한 진행을 위하여 작업을 진행하면서 사용 될 수 있는 내용을 정의 한다.
가이드에서 다루는 내용은 HTML, CSS, IMAGE, JS 등의 공통된 내용 및 ISSUE 사항을 다루며 향후 웹사이트의 디자인 관리 및 사이트 유지보수 개발의 지침서로서 배포하여 활용하는 데 그 목적이 있다. 개발자는 배포받은 가이드를 준수하여 HTML마크업 을 진행하며 ISSUE 사항이 발생하면 수정 및 프론트 개발자 에게 재배포를 원칙으로 한다.

HTML 정의

신규 페이지 생성시 DOCTYPE은 HTML5이며, 파일 인코딩은 UTF-8로 생성한다.
페이지 생성 및 유지보수 진행시 하단 코드를 참고하여 진행한다.
추가되는 css는 common.css 하단에 추가되는 해당 매뉴의 css 를 위치하며, javascript파일은 css 보다 하단에 위치한다. 
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>LAS : WEB STYLE GUIDE </title>
    <link rel="stylesheet" href="../../css/common.css">
    <script type="text/javascript" src="../../js/jquery-3.4.1.min.js"></script>
    <script type="text/javascript" src="../../js/common_ui.js"></script>
</head>
<body class=""><!-- 카테고리별 class를 body에 삽입한다. -->
<div id="wrap">
    <header id="header"></header>
    <main id="wrap"></main>
    <footer id="footer"></footer>
</div>
</body>
</html>

CSS 정의

CSS는 common.css 파일에 @import url(name.css) 호출을 한 후 프로젝트 최종 종료시 common.css 파일에 취합한다.
Library의 경우 common.css 파일보다 위에 위치한다. 
<link rel="stylesheet" href="../../css/library.css"><!--  해당하는 Library CSS 위치 -->
<link rel="stylesheet" href="../../css/common.css">

javascript 정의

UI 스크립트는 common_ui.js 파일에 작성하며, 라이브러리 사용의 경우 library.js해당 파일을 다운로드하여 삽입하여 사용한다.
<script type="text/javascript" src="../../js/library.js"></script>
<script type="text/javascript" src="../../js/common_ui.js"></script>

주석처리

마크업진행시 주석처리 정의 : 주석은 HTML 작업시 해당 section 구분마다 주석처리를 해주며, 수정및 ISSUE발생시 주석처리를 한다.
개발에 전달 해야할 ISSUE 혹은 수정시 발생시 Develop 약자인 [D] 의 약자 로 해당하는 부분에 내용과 시작 종료일자를 함께 작성 한다. 
HTML 기본 주석
<!-- 주석 시작 -->
<!-- // 주석 시작 -->
<!-- [D] 2019-XX-XX 수정내용 -->
<!-- // [D] 2019-XX-XX 수정내용 -->
CSS 주석 문구 작성시 마크업진행시 해당 메뉴의 네임을 작성하고, 종료 주석처리는 진행하지 않는다.
단, 이미진행이 된 부분의 CSS의 경우 수정날자를 기입한다. 
CSS 기본 주석
/* 메뉴 네임 */
/* 메뉴 2019-XX-XX 수정 */

네임 정의

사전에 예약어로 사용되는 네임을 제외한 HTML 네임은 개발자의 기준으로 선정하며 언더스코어" _ " 가 아닌 하이픈" - "으로 정의 하여 지정하며, IMAGE는 언더스코어" _ "로 정의 한다.
하이픈" - "는 2개 이상의 단어를 조합할때만 사용하며, 그이상의 단어 조삽시 최초 하이픈" - " 그 뒤에 단어는 Camel 기법(낙타)으로 진행한다. 
네이밍 사용 예시
<div class="guide-container">
<div class="guide-contentBox">
<img src="images_images" alt="이미지">

Devices 검수

Devices 검수는 제공되는 Devices를 대상으로 검수를 진행하도록 하며, 그 이외의 미제공 되는 Devices의 경우 브라우저에서 제공되는 해상도를 포함하여 검수진행한다.
